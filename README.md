Forensic analysis of the virtual machine memory dump (vmem) task 2 Holostih
# Анализ дампа памяти виртуальной машины

Проект для автоматического анализа дампа памяти (`var-2.vmem`) с использованием Volatility и генерации PDF‑отчёта с результатами.

## Функциональность

Проект выполняет анализ дампа памяти и отвечает на следующие вопросы:

1. Определяет семейство и версию ОС на компьютере Джона.
2. Находит процесс, установивший сетевое соединение с портом 64593.
3. Проверяет наличие сетевых соединений с локальными портами 135–140.
4. Указывает идентификаторы процессов для соединений из п. 3.
5. Находит идентификаторы родительских процессов.
6. Определяет все процессы, порождённые процессами из п. 3.
7. Подсчитывает количество запущенных процессов `svchost.exe` и их PID.
8. Указывает имена исполняемых файлов, запустивших эти процессы.
9. Выявляет признаки заражения среди процессов `svchost.exe`.
10. Находит полный путь к исполняемому файлу процесса с PID 1224.

Настройка
Параметры анализа можно изменить в config.py:

DUMP_FILE — путь к файлу дампа.

OUTPUT_DIR — папка для сохранения результатов.

TARGET_PORTS — порты для проверки.

PLUGINS — список плагинов Volatility для запуска.

Устранение неполадок
Ошибка «vol.py не найден»: убедитесь, что Volatility 3 установлен и добавлен в PATH.

Таймаут анализа: увеличьте значение timeout в analysis_engine.py для больших дампов.

Отсутствие данных в отчёте: проверьте, что файл дампа корректен и совместим с Volatility.

Ошибки парсинга: убедитесь, что версия Volatility совместима с форматом дампа.
